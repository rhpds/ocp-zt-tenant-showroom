= Module 1: Deploy a Workload on OpenShift
:imagesdir: ../assets/images

In this module you will create your personal namespace and deploy a ConfigMap and Deployment to it.

== Exercise 1: Create your namespace

First, confirm your username and create your namespace.

. Check your username:
+
[source,sh,role=execute]
----
oc whoami
----

. Create your personal namespace:
+
[source,sh,role=execute]
----
oc new-project zt-test-$(oc whoami)
----

. Set a variable so the remaining commands are shorter:
+
[source,sh,role=execute]
----
export NS=zt-test-$(oc whoami)
echo "Using namespace: $NS"
----

[NOTE]
====
If you see `Error from server (AlreadyExists)` the namespace already exists -- that is fine.
Run the `export NS=...` line above to set the variable and continue.
====

== Exercise 2: Explore your namespace in the OCP Console

. In the *OCP Console* tab, click the perspective switcher in the top-left and select *Developer*.

. Click the *Project* dropdown at the top and search for `zt-test`.
Select your namespace `zt-test-<your-username>`.

. Click *Topology* in the left navigation.
The view is empty for now -- you will deploy an application here in the next step.

== Exercise 3: Create the ConfigMap and Deployment

. Create the ConfigMap:
+
[source,sh,role=execute]
----
oc create configmap zt-test-config \
  --from-literal=env=test \
  -n $NS
----

. Create the Deployment:
+
[source,sh,role=execute]
----
oc create deployment zt-test-app \
  --image=registry.access.redhat.com/ubi9/nginx-122:latest \
  -n $NS
----

== Verify

Wait for the pod to reach *Running* status:

[source,sh,role=execute]
----
oc get pods -n $NS -l app=zt-test-app
----

.Expected output
[source,texinfo]
----
NAME                           READY   STATUS    RESTARTS   AGE
zt-test-app-xxxxxxxxx-xxxxx   1/1     Running   0          30s
----

You can also refresh the *OCP Console* Topology view -- `zt-test-app` will appear as a workload.

[TIP]
====
If the pod is in `Pending` or `ContainerCreating` state, wait 30 seconds and run the verify command again.
====
