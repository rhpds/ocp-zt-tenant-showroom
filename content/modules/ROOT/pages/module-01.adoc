= Module 1: Create a ConfigMap in Your Namespace

In this module create a ConfigMap in your namespace `{user}-zttest`.

== Your Task

[source,sh,role="execute send-to"]
----
oc create configmap lab-config \
  --from-literal=env=dev \
  --from-literal=owner={user} \
  -n {user}-zttest
----

=== Verify

[source,sh,role="execute send-to"]
----
oc get configmap lab-config -n {user}-zttest -o yaml
----

== How Validation Works

The `validation.yml` playbook:

[source,yaml]
----
- hosts: localhost
  connection: local
  gather_facts: false
  tasks:
    - kubernetes.core.k8s_info:
        kubeconfig: "{{ k8s_kubeconfig | default(omit) }}"
        validate_certs: false
        api_version: v1
        kind: ConfigMap
        name: lab-config
        namespace: "{{ student_ns }}"
      register: r_cm

    - ansible.builtin.set_fact:
        _task1_ok: "{{ r_cm.resources | length > 0 }}"
        _task2_ok: >-
          {{ r_cm.resources | length > 0 and
             r_cm.resources[0].data is defined and
             r_cm.resources[0].data.get('env','') == 'dev' }}

    - validation_check:
        check: "{{ _task1_ok and _task2_ok }}"
        pass_msg: |
          ✅ Task 1: ConfigMap lab-config exists in {{ student_ns }}
          ✅ Task 2: ConfigMap has env=dev
        error_msg: |
          {{ '✅' if _task1_ok else '❌' }} Task 1: ConfigMap lab-config exists
          {{ '✅' if _task2_ok else '❌' }} Task 2: ConfigMap has env=dev

          Fix:
            oc create configmap lab-config --from-literal=env=dev -n {{ student_ns }}
----
