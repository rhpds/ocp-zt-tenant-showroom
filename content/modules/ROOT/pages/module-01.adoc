= Module 1: Deploy a Workload on OpenShift
:imagesdir: ../assets/images

In this module you will deploy a ConfigMap and a Deployment to your personal namespace.
Your namespace has been pre-created and is ready to use.

== Exercise 1: Find your namespace in the OCP Console

. In the *OCP Console* tab, click the perspective switcher in the top-left and select *Developer*.

. Click the *Project* dropdown at the top and search for `zt-test`.
Select your namespace — it follows the pattern `zt-test-<your-username>`.

. Click *Topology* in the left navigation.
The view is empty for now — you will deploy an application here in the next exercise.

[NOTE]
====
Your username in this lab is your OCP login name.
You can confirm it by checking *User Management* → *Users* in the Administrator perspective,
or by looking at the project name that was pre-assigned to you.
====

== Exercise 2: Create the ConfigMap and Deployment

Set your namespace as a variable so the remaining commands are shorter.
Replace `<username>` with your OCP username.

. Set the namespace variable:
+
[source,sh,role=execute]
----
export NS=zt-test-$(oc whoami)
echo "Using namespace: $NS"
----

. Create the ConfigMap:
+
[source,sh,role=execute]
----
oc create configmap zt-test-config \
  --from-literal=env=test \
  -n $NS
----

. Create the Deployment:
+
[source,sh,role=execute]
----
oc create deployment zt-test-app \
  --image=registry.access.redhat.com/ubi9/nginx-122:latest \
  -n $NS
----

== Verify

Wait for the pod to reach *Running* status:

[source,sh,role=execute]
----
oc get pods -n $NS -l app=zt-test-app
----

.Expected output
[source,texinfo]
----
NAME                           READY   STATUS    RESTARTS   AGE
zt-test-app-xxxxxxxxx-xxxxx   1/1     Running   0          30s
----

You can also check the *OCP Console* Topology view — `zt-test-app` will appear as a workload.

[TIP]
====
If the pod is in `Pending` or `ContainerCreating` state, wait 30 seconds and run the verify command again.
====
