= Module 2: Apply a Compliance Label
:imagesdir: ../assets/images

In this module you will apply a compliance label to your namespace.
This simulates a simple policy enforcement step -- a common pattern in real ZT labs.

== Exercise 1: View namespace labels in the OCP Console

. In the *OCP Console* tab, switch to the *Administrator* perspective using the
top-left switcher.

. In the left navigation, go to *Home* -> *Projects*.

. Find your namespace (`zt-test-<your-username>`) and click it.

. On the *Details* tab, scroll down to the *Labels* section.
Note the current labels -- `compliance=passed` is not there yet.

== Exercise 2: Apply the compliance label

. Set your namespace variable (if your terminal session restarted):
+
[source,sh,role=execute]
----
export NS=zt-test-$(oc whoami)
----

. Apply the compliance label:
+
[source,sh,role=execute]
----
oc label namespace $NS compliance=passed --overwrite
----

== Verify

Confirm the label is set:

[source,sh,role=execute]
----
oc get namespace $NS --show-labels
----

.Expected output
[source,texinfo]
----
NAME               STATUS   AGE   LABELS
zt-test-student1   Active   10m   compliance=passed,kubernetes.io/metadata.name=zt-test-student1,...
----

You can also refresh the *OCP Console* Projects view -- the `compliance=passed` label
will now appear in the Labels section of your namespace details.

[TIP]
====
If you see `compliance=failed` or no compliance label, re-run the `oc label` command
above and verify again.
====
