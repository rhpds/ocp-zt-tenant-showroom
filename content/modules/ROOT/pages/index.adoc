== How This Lab Works

Each module has two buttons at the bottom of the page:

🚀 *Solve Module*::
Automatically completes all tasks in the module for you.
Use this if you are stuck or want to skip ahead.
Output streams live in a terminal panel so you can see what's happening.

✓ *Validate Module*::
Runs automated checks against your namespace to verify you completed the tasks correctly.
Shows ✅ for each passing check and ❌ with a fix hint for failing ones.

=== Step-by-step

. Follow the instructions in the module
. Run the commands in the *Terminal* tab on the right
. Click *✓ Validate Module* to check your work
. If it fails, fix the issue and validate again
. If stuck, click *🚀 Solve Module* — then validate to confirm it passed

== Writing Playbooks (Lab Author Guide)

=== Available Extravars

These variables are automatically injected into every playbook by the runner:

[cols="1,2,1",options="header"]
|===
| Variable | Description | Example
| `student_ns` | Student's primary namespace | `devuser-abc-zttest`
| `student_ns2` | Student's second namespace | `devuser-abc-ztworkspace`
| `student_user` | Student username | `devuser-abc`
| `guid` | Lab GUID | `abc123`
| `k8s_kubeconfig` | Path to kubeconfig file with namespace-admin access | `/tmp/zt-runner-xxx.kubeconfig`
|===

=== validate.yml Template

[source,yaml]
----
---
- name: Module 1 Validation
  hosts: localhost
  connection: local
  gather_facts: false
  tasks:
    - name: Check ConfigMap exists
      kubernetes.core.k8s_info:
        kubeconfig: "{{ k8s_kubeconfig | default(omit) }}"
        validate_certs: false
        api_version: v1
        kind: ConfigMap
        name: my-config
        namespace: "{{ student_ns }}"
      register: r_cm

    - name: Validate
      validation_check:
        check: "{{ r_cm.resources | length > 0 }}"
        pass_msg: "✅ ConfigMap exists in {{ student_ns }}"
        error_msg: |
          ❌ ConfigMap not found in {{ student_ns }}

          Fix:
            oc create configmap my-config -n {{ student_ns }}
----

=== solve.yml Template

[source,yaml]
----
---
- name: Module 1 Solve
  hosts: localhost
  connection: local
  gather_facts: false
  tasks:
    - name: Create ConfigMap
      kubernetes.core.k8s:
        kubeconfig: "{{ k8s_kubeconfig | default(omit) }}"
        validate_certs: false
        state: present
        definition:
          apiVersion: v1
          kind: ConfigMap
          metadata:
            name: my-config
            namespace: "{{ student_ns }}"
          data:
            key: value
----

=== Optional: oc login as Student Keycloak User

If your validation needs to check something as the *student's Keycloak identity* (not the SA token), you can use `oc login` directly from the runner container. The runner has the `oc` CLI installed and can reach the OCP API.

This does NOT change the default behavior — the kubeconfig SA is still used for all other tasks.

[source,yaml]
----
---
- name: Validate — check as student Keycloak user
  hosts: localhost
  connection: local
  gather_facts: false
  tasks:
    # Default: use SA kubeconfig for most checks
    - name: Check ConfigMap exists (as SA)
      kubernetes.core.k8s_info:
        kubeconfig: "{{ k8s_kubeconfig | default(omit) }}"
        validate_certs: false
        api_version: v1
        kind: ConfigMap
        name: my-config
        namespace: "{{ student_ns }}"
      register: r_cm

    # Optional: login as Keycloak user for identity-specific checks
    - name: Login as student Keycloak user
      when:
        - password is defined
        - openshift_api_url is defined
      ansible.builtin.command:
        cmd: >
          oc login {{ openshift_api_url }}
          -u {{ student_user }}
          -p {{ password }}
          --insecure-skip-tls-verify=true
      changed_when: false
      no_log: true

    - name: Check RoleBinding as student identity
      when: password is defined
      ansible.builtin.command:
        cmd: oc get rolebinding -n {{ student_ns }}
      register: r_rb
      changed_when: false

    - name: Validate
      validation_check:
        check: "{{ r_cm.resources | length > 0 }}"
        pass_msg: "✅ ConfigMap exists"
        error_msg: "❌ ConfigMap not found"
----

NOTE: `password` is available when the showroom role adds it to `user_data.yml`. Check your AgV catalog's `user_data` section to confirm it is included.

=== Rules

* Always use `validation_check` — never `ansible.builtin.fail`
* Always pass `kubeconfig: "{{ k8s_kubeconfig | default(omit) }}"` to `kubernetes.core` tasks
* Use `student_ns` not hardcoded namespace names
* The `job_info_dir` variable is injected automatically — do not define it in your playbook
* Use `when: password is defined` to guard optional `oc login` so default behavior is preserved
