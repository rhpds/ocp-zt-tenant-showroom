== How Solve/Validate Works

The `zt-runner` sidecar container reads the `showroom-userdata` ConfigMap and injects extravars into every Ansible playbook:

[cols="1,2,1",options="header"]
|===
| Extravar | Source | Example value
| `student_user` | `user` field in CM | `devuser-abc123`
| `student_ns` | derived `{user}-zttest` | `devuser-abc123-zttest`
| `student_ns2` | derived `{user}-ztworkspace` | `devuser-abc123-ztworkspace`
| `k8s_kubeconfig` | zt-runner-kubeconfig Secret | `/tmp/zt-runner-xxx.kubeconfig`
| `guid` | `guid` field in CM | `abc123`
|===

== API Endpoints (Flask SSE)

[source,bash]
----
# Check which modules/stages are available
curl -sk $SHOWROOM/runner/api/config

# Trigger solve (returns Job_id for polling)
curl -sk -X POST $SHOWROOM/runner/api/module-01/solve

# Stream solve output via SSE
curl -sk $SHOWROOM/stream/solve/module-01

# Poll job result
curl -sk $SHOWROOM/runner/api/job/{Job_id}
----

Click *Next* to start Module 1.
