== How the runner works

The ZT validation runs inside a `zerotouch-automation` container (runner) co-located in the showroom pod.

* Runner image: `ghcr.io/prakhar1985/ocp-zt-runner:latest`
* Based on `quay.io/rhpds/zerotouch-automation:v1.2.0` with `kubernetes.core` pre-installed
* Communicates with OCP via the pod's service account token — no `oc` binary needed
* Exposes a FastAPI on port 8501, routed through nginx at `/runner/`
* Playbooks live in `runtime-automation/` in this repo
